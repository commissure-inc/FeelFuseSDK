Interface Hierarchy

Core Management Interfaces

  • IHapticManager: Basic device management interface
  • IHapticManager: Generic haptic data flow management
  • IHapticHands: Hand positioning and control interface
  • IHapticHands: Generic hand-based haptic input access

Input/Output Interfaces

  • IHapticInput: Haptic input sources management
  • IHapticOutput: Haptic output value distribution
  • IHapticSource: Individual haptic data source
  • IValueCollector: Value collection utility interface

Device Interfaces

  • IHapticDevice: Base haptic device interface
  • IHapticInputRegisterer: Input registration management
  • IHapticSourceRegisterer: Source registration management

Body Part Model Interfaces

  • IBodyPartModel: Base body part model interface
  • IHandModel: Hand-specific model interface
  • IHandTrackingModel: Extended hand tracking model
  • IHandModelsContainer: Hand model container management

Key Value Types

BodyPartTarget

A sophisticated body part targeting system using hierarchical byte values.

Module: Commissure.ModelTracking.Core Location: Assets/Commissure/ModelTracking/Core/Scripts/Value.cs

The BodyPartTarget struct provides precise hierarchical identification of body parts for haptic feedback targeting:

Static Properties:
- Unknown, All (basic)
- Upper, Lower, Right, Left, Front, Back (directions)
- Arm, Head, Chest, Waist, Leg (major body parts)
- Upperarm, Forearm, Hand, etc. (detailed parts)
- Palm, Finger, Sole, Toe (extremities)
- Thumb, Index, Middle, Ring, Pinky (finger specification)
- FirstPhalanx, SecondPhalanx, ThirdPhalanx (finger segments)

Design Patterns

Interface Segregation

Interfaces are carefully segregated by responsibility:

  • Device management vs. data flow management
  • Basic vs. generic type-safe operations
  • Input vs. output concerns

Generic Type Safety

The <TValue> parameter allows type-safe haptic data processing while maintaining flexibility for different data types (audio, force, etc.).

Observer Pattern

UnityEvent-based notifications for:

  • Model rebuilding events
  • Container lifecycle events
  • Registration/unregistration events

Strategy Pattern

Different implementations can be plugged in through interfaces:

  • Mock vs. real device implementations
  • Meta vs. Unity XR platform implementations
  • V0 vs. V1 device versions

Relationship Types

  • Inheritance (--|>): Interface extension relationships
  • Composition (-->): Ownership and management relationships
  • Association: Usage and dependency relationships

Interface Color Coding

  • Blue (Interface Class): All interface definitions
  • Orange (Value Class): Concrete value types like BodyPartTarget
