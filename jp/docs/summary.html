FeelFuseSDK ソースコード群の概要

システム全体の概要

FeelFuseSDKの触覚フィードバックシステムは、5つの主要なコンポーネント群で構成されています。 これらは協調して、ゲームイベントや物理インタラクションから物理的な触覚デバイスへのデータフローを実現します。

データフロー

[ゲームイベント/物理演算]
        ↓
[HapticSource群] ← [HapticRegisterer群] → [HapticInput群]
        ↓                                        ↓
        └────→ [HapticManager] ─────────────────┘
                    ↓
            [HapticOutput群]
                    ↓
            [HapticDevice群]
                    ↓
          [物理触覚デバイス]

1. HapticSource群

役割: 触覚フィードバックデータの生成源

含まれるクラス

  • HapticAudioSource
  • HapticForceSource

機能

  • ゲームイベントや物理インタラクションから触覚データを生成
  • オーディオベース（音声クリップ、音量、ピッチ）またはフォースベース（位置、方向、強度）のフィードバックを提供
  • CollectValues()メソッドを通じて、HapticInputにデータを送信
  • HapticRegisterer群を介してHapticInputに動的に登録・解除される

2. HapticInput群

役割: 特定の身体部位に対する触覚データの集約点

含まれるクラス

  • HapticAudioInput
  • HapticForceInput

機能

  • 特定のBodyPartTarget（左手、右手など）に対応
  • 複数のHapticSourceからのデータを収集・管理
  • 登録されたSourceのリストを保持
  • HapticRegistererからの登録・解除リクエストを管理
  • HapticManagerに対してデータを提供

3. HapticRegisterer群

役割: SourceとInputの動的な接続管理

含まれるクラス

汎用Registerer

  • HapticTriggerInputRegisterer
  • HapticTriggerSourceRegisterer
  • HapticSequentialSourceRegister

プラットフォーム固有Registerer（Meta XR Interaction統合）

  • HapticMetaInteractionInputRegisterer
  • HapticMetaInteractionSourceRegisterer

プラットフォーム固有Registerer（Unity XR Interaction Toolkit統合）

  • HapticXRITInteractionInputRegisterer
  • HapticXRITInteractionSourceRegisterer

機能

  • ゲームロジックに基づいて、いつ、どのようにSourceがInputに接続するかを制御
  • 異なる登録戦略を実装:
    • Trigger系: コライダーの衝突検出に基づく自動登録・解除
    • Sequential系: 順次的なソースの切り替え管理
    • MetaInteraction系: Meta XR Interaction SDK（IInteractorView/IInteractableView）との統合
      • InputRegisterer: Meta InteractorをHapticInputに関連付け
      • SourceRegisterer: Meta Interactableの選択イベント（WhenSelectingInteractorViewAdded/Removed）でSourceを登録・解除
    • XRITInteraction系: Unity XR Interaction Toolkit（XRBaseInteractor/XRBaseInteractable）との統合
      • InputRegisterer: XRIT InteractorをHapticInputに関連付け
      • SourceRegisterer: XRIT Interactableの選択イベント（selectEntered/selectExited）でSourceを登録・解除
  • Mediatorパターンを使用し、Source-Input間の柔軟な接続を実現
  • プラットフォーム固有のインタラクションシステムとシームレスに統合

4. HapticOutput群

役割: デバイス向けのデータ処理・変換層

含まれるクラス

  • HapticAudioOutputV0 / HapticAudioOutputV1
  • HapticForceOutputV0 / HapticForceOutputV1

機能

  • HapticManagerから収集された触覚データを受信
  • BodyPartTargetによるフィルタリング（左手/右手の区別など）
  • デバイス固有の形式への変換:
    • Force系: ForceConverterを使用した角度変換、ガウシアンフィルタ適用
    • Audio系: オーディオクリップの処理、再生モード管理（OneShot、Loop、Stop）
  • V0とV1でハードウェアバージョンの違いに対応

5. HapticDevice群

役割: ハードウェア通信レイヤー

含まれるクラス

  • HapticDeviceV0
  • HapticDeviceV1

機能

  • すべてのOutputを統括するトップレベルマネージャー
  • 物理的な触覚デバイスとの通信を担当
  • V0: シリアル通信（COMポート）を使用
  • V1: OSC（Open Sound Control）プロトコルをネットワーク経由で使用
  • 処理済みデータを最終的に物理ハードウェアに送信

コンポーネント間の関係性

データフローの詳細（各フレーム）

  1. BeginFrame: HapticManagerがフレーム開始を全コンポーネントに通知

  2. CollectValuesフェーズ:

    • Managerが保留中のSource登録をInputに承認
    • 各Inputの登録済みSourceからCollectValues()を呼び出し
    • 触覚データがValueCollectorに集約される

  3. DistributeValuesフェーズ:

    • 収集されたデータをすべてのOutputに配信
    • 各OutputがBodyPartTargetでフィルタリングして保存

  4. OnValuesUpdatedフェーズ:

    • HapticDeviceに通知
    • Deviceが各Outputを呼び出してデータ処理（力→角度変換、音声処理）
    • 最終コマンドをハードウェアに送信（シリアル/OSC）

  5. EndFrame: クリーンアップと次フレームの準備

多重度の関係

  • Source → Input: 多対多（Registererが管理）
  • Input → Manager: 多対1（登録関係）
  • Manager → Output: 1対多（配信関係）
  • Output → Device: 多対1（集約関係）
