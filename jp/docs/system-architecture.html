Architecture Layers

1. Infrastructure Layer

  • Unity 2022.3.62f1: Core game engine
  • Meta XR SDK 78.0.0: Meta Quest platform support
  • OpenXR 1.14.3: Cross-platform XR standard

2. Platform Abstraction Layer

  • Hands.Meta/XRIT: Hand tracking implementations
  • Interaction.Meta/XRIT: Interaction system implementations
  • Device V0/V1/Mock: Device abstraction versions

3. Core Layer

  • HapticManager: Central system coordinator
  • HapticHands: Hand position and targeting control
  • Input/Output System: Haptic data flow management

4. Application Layer

  • Test Scenes: Development and validation environments
  • Test Prefabs: Reusable haptic components

Key Design Patterns

Interface-Based Design

All major components are interface-driven for flexibility and testability.

Generic Type System

Type-safe haptic value processing through generic interfaces like IHapticManager<TValue>.

Registrar Pattern

Dynamic component registration/deregistration via IHapticSourceRegisterer.

Layered Architecture

Clear separation of concerns across infrastructure, platform, core, and application layers.

Project Structure

The FeelFuseSDK is organized into three separate Unity projects to support different deployment scenarios:

1. FeelFuseSDK-core

Purpose: Core SDK development and shared assets Location: FeelFuseSDK-dev/FeelFuseSDK-core/ Contents:

  • All core modules (Base, HapticsSystem, ModelTracking)
  • Platform-agnostic implementations
  • Shared scripts and assets
  • Development and testing environment

2. FeelFuseSDK-meta

Purpose: Meta Quest optimized builds Location: FeelFuseSDK-dev/FeelFuseSDK-meta/ Target Platform: Android (Meta Quest devices) Features:

  • Meta XR SDK integration
  • Meta-specific hand tracking (Hands_Meta)
  • Meta interaction system (Interaction_Meta)
  • Optimized for Quest performance

3. FeelFuseSDK-xrit

Purpose: Cross-platform XR builds Location: FeelFuseSDK-dev/FeelFuseSDK-xrit/ Target Platform: OpenXR compatible devices Features:

  • Unity XR Interaction Toolkit integration
  • Unity XR Hands implementation (Hands_XRIT)
  • Unity XR interaction system (Interaction_XRIT)
  • Broader device compatibility

Project Selection Guide

  • Use FeelFuseSDK-core for SDK development and testing
  • Use FeelFuseSDK-meta for Meta Quest-specific applications requiring maximum performance
  • Use FeelFuseSDK-xrit for applications targeting multiple XR platforms or non-Meta devices
