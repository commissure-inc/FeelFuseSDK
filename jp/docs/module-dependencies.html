Module Categories

Foundation Module (Green)

  • Base: Foundation module with no dependencies, provides common utilities, registries, and shared interfaces

Core Modules (Blue)

  • HapticsSystem.Core: Main haptic system interfaces and management, depends on Base
  • ModelTracking.Core: Body part tracking and BodyPartTarget system, depends on Base

Device Modules (Purple)

  • Device: Base device abstraction, depends on HapticsSystem.Core and ModelTracking.Core
  • Device.V0: Legacy device implementation
  • Device.V1: Current device implementation with OSC communication
  • Both V0 and V1 depend on Device, both Core modules, and Base

Platform Modules (Orange)

  • ModelTracking.Hands_Meta: Meta hand tracking implementation
  • ModelTracking.Hands_XRIT: Unity XR hand tracking implementation
  • HapticsSystem.Interaction_Meta: Meta interaction system implementation
  • HapticsSystem.Interaction_XRIT: Unity XR interaction implementation

External Dependencies (Light Green)

  • Meta XR Core: Meta Quest platform SDK
  • Meta XR Interaction: Meta interaction framework
  • Unity XR Hands: Unity's cross-platform hand tracking

Conditional Compilation

Meta Platform Defines

  • Meta_XR_Hands: Enabled when Meta XR Core package is present
  • Meta_XR_Interaction: Enabled when Meta Interaction package is present

Unity Platform Defines

  • Unity_XR_Hands: Enabled when Unity XR Hands package is present

Dependency Rules

  1. Base is the foundation - no external dependencies, provides common infrastructure
  2. Core modules (HapticsSystem.Core, ModelTracking.Core) build on Base
  3. Device layer depends on both Core modules (HapticsSystem and ModelTracking)
  4. Platform adapters depend on Base, Core modules, plus external SDKs
  5. Version modules (V0, V1) extend the base Device module and depend on all foundation layers
