Platform Specific Implementations

Meta Quest Platform

Advantages:

  • Native hand tracking with high precision
  • Optimized for Meta hardware
  • Direct access to Meta interaction features
  • Gesture recognition support

Components:

  • Hands.Meta: Direct integration with Meta Hand Tracking API
  • Interaction.Meta: Meta Interaction SDK integration
  • Device.V1: Full haptic device support

OpenXR Compatible Devices

Advantages:

  • Cross-platform compatibility
  • Standardized API across devices
  • Unity's official XR support
  • Future-proof architecture

Components:

  • Hands.XRIT: Unity XR Hands implementation
  • Interaction.XRIT: XR Interaction Toolkit integration
  • Device.V1: Universal haptic device support

Standalone PC / Development

Advantages:

  • No hardware requirements
  • Editor-based testing
  • Rapid development cycle
  • Full feature simulation

Components:

  • Device.Mock: Simulated haptic devices
  • Both Meta and XRIT implementations available
  • Complete development environment

Conditional Compilation System

Meta Platform Defines

#if Meta_XR_Hands
    // Meta hand tracking specific code
#endif

#if Meta_XR_Interaction
    // Meta interaction specific code
#endif

Triggered by:

  • com.meta.xr.sdk.core package presence
  • com.meta.xr.sdk.interaction package presence

Unity Platform Defines

#if Unity_XR_Hands
    // Unity XR Hands specific code
#endif

Triggered by:

  • com.unity.xr.hands package presence

Device Version Strategy

Device V1 (Current)

  • ✅ Full feature implementation
  • ✅ Performance optimized
  • ✅ Cross-platform support
  • ✅ Recommended for new projects

Device V0 (Legacy)

  • ⚠️ Limited feature set
  • ⚠️ Legacy compatibility only
  • ⚠️ May be deprecated in future

Device Mock (Development)

  • ✅ No hardware required
  • ✅ Editor support
  • ✅ Complete API simulation
  • ✅ Ideal for development/testing

Platform Selection Guidelines

Choose Meta Implementation When:

  • Targeting Meta Quest exclusively
  • Requiring maximum performance
  • Using advanced hand gestures
  • Leveraging Meta-specific features

Choose Unity XR Implementation When:

  • Supporting multiple VR platforms
  • Using Unity's standard XR pipeline
  • Requiring cross-platform compatibility
  • Future-proofing for new devices

Use Mock Implementation For:

  • Development without hardware
  • Automated testing
  • Editor-based prototyping
  • Continuous integration pipelines

Package Dependencies

Required Packages

  • Unity 2022.3.62f1 or higher
  • Unity XR Management 4.5.2

Optional Platform Packages

  • Meta Platform: Meta XR SDK 78.0.0
  • OpenXR Platform: OpenXR 1.14.3
  • Cross-platform: Unity XR Hands (when available)

Development Packages

  • Unity Test Framework (for testing)
  • Unity Memory Profiler (for optimization)
